/>

Use a showcase block when you want readers to compare two or more approaches quickly.

<ComponentShowcase
  title="Common docs pairings"
  description="These grouped patterns work well for product docs, internal playbooks, and onboarding guides."
>
  <ShowcaseItem label="Checklist">
    Keep preflight tasks in one card so setup requirements do not get buried inside the main flow.
  </ShowcaseItem>
  <ShowcaseItem label="Example Copy">
    Show a sample announcement, policy snippet, or launch note beside the explanation that supports it.
  </ShowcaseItem>
  <ShowcaseItem label="Do This">
    Standardize on short, action-oriented language when documenting repeatable editorial workflows.
  </ShowcaseItem>
  <ShowcaseItem label="Avoid This">
    Call out patterns that create maintenance debt, like vague article titles or category slugs that drift from product language.
  </ShowcaseItem>
</ComponentShowcase>

<Callout title="A good default" tone="info">
  If you are unsure whether a section deserves a showcase, ask whether readers need to compare options. If yes, the component usually earns its space.
</Callout>
