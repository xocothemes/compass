Every article in Compass is just a Markdown or MDX file with a little frontmatter. Once you add one document and connect it to a category, the homepage, sidebar, and article route all update automatically.

<Callout tone="success">
  Keep simple guides in Markdown. Reach for MDX when a page needs richer callouts, steps, tabs, accordions, or structured presentation beyond standard prose.
</Callout>

## Start with frontmatter

Use frontmatter to define how the article is listed:

```mdx
---
title: "Deploy Compass"
description: "Publish your docs site to your hosting platform."
category: "docs"
order: 5
updatedAt: 2026-06-03
---
```

## Add the body content

Below the frontmatter, write normal Markdown or MDX content:

```mdx
## Publish checklist

- Confirm `siteUrl`
- Run `npm run build`
- Verify links and metadata
```

## Where it shows up

Once saved, the article will appear in:

- the category page for its `category`
- the sidebar for that section
- the direct article route generated from the filename

## MDX when you need more than Markdown

Most docs can stay plain Markdown, but MDX is helpful when you want:

- custom components
- richer callouts
- buttons, steps, tabs, or accordions
- embedded demos or previews
